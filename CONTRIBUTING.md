# Contributing to hardFork

## Getting Started

1. Clone the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test thoroughly
5. Submit a pull request

## Code Standards

- All Clarity contracts must be well-documented
- Include unit tests for all new functionality
- Follow Clarity best practices and security guidelines
- Document any external dependencies

## Commit Guidelines

- Use clear, descriptive commit messages
- Reference GitHub issues when applicable
- Keep commits atomic and focused

## Testing

- Write tests for all smart contract functionality
- Run the test suite before submitting PRs
- Verify contracts compile without warnings in Clarinet

## Pull Request Process

1. Ensure your branch is up to date with main
2. Update documentation as needed
3. Provide a clear description of changes
4. Request review from team members
5. Address feedback before merging

## Questions or Issues?

Open a GitHub issue to discuss any problems or questions.
